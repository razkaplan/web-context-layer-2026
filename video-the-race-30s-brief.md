# THE RACE — 30-Second Video Brief

**Format:** Split-screen motion graphics. Fast cuts. No voiceover narration. Text on screen + sound design only.
**Tone:** Tense. Competitive. The viewer should feel the pressure.
**Aspect:** 16:9 (LinkedIn/YouTube) + 9:16 vertical cut (Reels/Shorts)
**Music:** Electronic pulse. Starts slow, builds. Think countdown clock tension.

---

## FULL GENERATION PROMPT

Use this prompt with video generation tools (Runway Gen-4, Kling, Pika, or a motion graphics brief for a designer):

---

**PROMPT:**

Create a 30-second split-screen motion graphics video. Dark background (#091B36). No voiceover. Text and sound design only. Electronic pulse soundtrack that builds from a single heartbeat to rapid percussive beats.

The video tells the story of two companies racing to launch the same data product. Left side (red-tinted, labeled "BUILD IT YOURSELF") shows a company building scraping infrastructure from scratch. Right side (blue-tinted, labeled "ONE API CALL") shows a company using a managed web data API.

**Visual style:** Clean, dark, tech-corporate. Think Stripe or Linear product videos. Sharp typography (Inter or similar geometric sans-serif). Monospaced code snippets. Flat color data visualizations. No 3D. No characters with faces. Use abstract representations: terminals, dashboards, code blocks, charts, notification badges, world maps.

**Pacing:** Fast. No shot longer than 3 seconds. Most shots 1-2 seconds. The left side gets progressively more chaotic (red alerts, stacking error messages, growing cost counters). The right side stays clean and calm (blue accents, smooth growth charts, green status indicators).

**Sequence:**

SHOT 1 (0-1s): Black screen. "Same market. Same idea." fades in white, centered.

SHOT 2 (1-2s): Hard vertical split. Left labeled "BUILD IT YOURSELF" in red. Right labeled "ONE API CALL" in blue. Visual: starting line graphic.

SHOTS 3-4 (2-6s): WEEK 1 to WEEK 4. Left: terminal with scrolling code, Kubernetes YAML, proxy configs, a Jira board filling up. Right: minimal code editor showing a 3-line API call returning clean JSON, then a customer dashboard with a growth line going up. Time labels overlay each side.

SHOT 5 (6-9s): MONTH 2. THE BREAK. Left side goes RED. "403 FORBIDDEN" error screens. Slack-style notifications flooding in: "scraper down", "feeds broken", "CEO asking". Right side stays calm: a dashboard reading "99.7% UPTIME" with a flat green line.

SHOT 6 (9-12s): MONTH 3. Left: cost counter climbing past $218K with revenue at $0. IP ban list scrolling. Right: revenue counter at $180K, 50 customer logos appearing, team whiteboarding new features.

SHOT 7 (12-15s): MONTH 6. Left: "3 ENGINEERS. FULL-TIME MAINTENANCE." Cost: $618K/yr. Uptime gauge stuck at 85%. Right: same 3 engineers building product. Cost: $12K/yr. Uptime: 99.99%. Enterprise logos fading in.

SHOT 8 (15-18s): Split collapses to center. Comparison table animates in, row by row, each row hitting with a beat: Time to market (6mo vs 2wk), Annual cost ($618K vs $12K), Uptime (85% vs 99.99%), Engineers on infra (3 vs 0).

SHOTS 9-10 (18-22s): Black screen. Large white text types out: "The question isn't whether to build a data product." Pause. "It's where you burn your engineering hours."

SHOT 11 (22-26s): Rapid montage, 0.5s per cut: "400M+ residential IPs" over world map nodes lighting up. "195 countries" with fully lit map. "99.7% success rate" with green checkmarks. "SOC 2 · ISO 27001 · GDPR · CCPA · CSA STAR" with certification badges.

SHOT 12 (26-30s): Clean frame. Bright Data logo (white). Tagline: "The collection layer for data products." URL: brightdata.com/data-products. Final heartbeat. Black.

**Color palette:** Background #091B36. Left side accent: #FF4444 (red, danger). Right side accent: #3D7FFC (blue, Bright Data). Success green: #22C55E. Text: white (#FFFFFF) and light gray (#94A3B8).

**Typography:** Headlines in bold geometric sans (Inter Black or similar). Data/code in monospace (JetBrains Mono). Numbers large and prominent. All caps for labels.

**Key emotion:** The viewer should feel the anxiety of the left side and the calm confidence of the right side. The contrast does the selling. No need for a narrator to explain. The visuals make it obvious.

## PRODUCTION NOTES

- **For AI video tools (Runway/Kling):** Generate each shot separately. The split-screen format is best assembled in post. Generate left-side and right-side clips independently and composite. Use the prompt above as a shot list.
- **For motion designers:** This is a 12-shot brief. Each shot has clear visual direction. The designer has creative freedom on transitions, easing, and micro-animations. The pacing and content are locked.
- **For LinkedIn cut (15s):** Use shots 5-8 only (the break, the bleed, the gap, the numbers). This is the highest-tension segment and works standalone.
- **For vertical (9:16):** Stack instead of split. Top = "Build" side, bottom = "API" side. Same content, vertical layout.
- **Estimated production:** $3,000-$8,000 for motion graphics. $500-$1,500 if using AI video tools with post-production compositing.
